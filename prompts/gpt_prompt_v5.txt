====================================================

Segmentation priority MUST follow semantic reasoning moves,

RULE S2. Philosophy vs Capability Separation
If a unit contains:
- coexistence / philosophy / principle explanation
AND
- description of human capability or traits
they MUST be split into separate units.

RULE S3. Transition Anchor Isolation
If a sentence begins with a strong transition marker
(Therefore, However, Nevertheless, On the other hand,
First, Second, Finally, In conclusion),
that sentence MUST start a new unit.

RULE S4. Final Recommendation Isolation
The final recommendation sentence (MAIN CLAIM anchor)
MUST be its own unit or part of the final unit block.

RULE S5. Target Unit Density
Preferred number of units = 5–8.
If segmentation results in fewer than 5 units,
you MUST re-split using S1–S4 until at least 5 units exist.

RULE S6. Semantic Move Priority Order
Priority for splitting:

1. Transition marker
2. Example introduction
3. Concept shift
4. Recommendation sentence
5. Paragraph boundary

Paragraph boundaries alone are NOT sufficient to prevent splitting.

RULE S7. Conservative Expansion Bias
When ambiguous between fewer or more units,
choose MORE units.

This rule overrides lower-bias counting rules.

COUNTING RULES (MINIMAL SET)
====================================================

A) layer_0.claims (int)
Count explicit assertions that present a proposition as true.
Heuristic (strict):
- Count sentences that make a declarative claim about the world or what should be done.
- Do not count pure examples unless they assert a general proposition.
If uncertain, undercount.

B) layer_0.evidence (int)
Count explicit evidence items only.
Accepted as evidence if the text explicitly provides:
- a concrete example used to support a claim
- a cited source, statistics, data, or specific observed instance
If the example is just illustrative without support intent, do not count.
If uncertain, undercount.

C) layer_0.per_unit.transitions (int array)
This is NOT “any connector word”.
Set transitions[i] as:
- transitions[0] = 0 always
- for i > 0:
  set to 1 ONLY if unit i begins with a strong structural transition marker that changes the reasoning step,
  such as: "Therefore", "However", "Nevertheless", "On the other hand", "In conclusion", "First", "Second", "Finally"
  Exclude paraphrase or rewording markers like "In other words".
If uncertain, output 0.

The transitions array length MUST equal the number of internally segmented units.

D) layer_1 counts (int)
- sub_claims: explicit supporting claims under a main claim
- warrants: explicit bridging principles (why the reason supports the claim)
- counterpoints: explicit opposing views or objections
- refutations: explicit rebuttals to counterpoints
Strict undercounting: if not explicit, output 0.
structure_type must be null (do not infer).

E) layer_2 (minimal)
- transitions: total number of structural transitions across the whole text.
  Use the same whitelist rule as per_unit.transitions.
- transition_ok: count of transitions that are logically coherent (<= transitions).
  If uncertain, set transition_ok = transitions.
- transition_types: always [] in this schema (do not output strings).
- belief_change: true only if the author explicitly changes their stance (e.g., "I used to think X but now Y").
  Otherwise false.

F) layer_3 (minimal)
- intent_markers: explicit intent phrases like "I will", "my purpose is", "this essay aims to"
- drift_segments: explicit statements showing topic drift or losing track ("anyway", "back to", "not sure", etc.)
- loops: explicit repetition loops ("as I said", repeated same point as correction loop)
- self_regulation_signals: explicit self-monitoring or self-correction phrases ("I revise", "I correct", "I realize my mistake")
If uncertain, output 0.
